# 2026-07-21 — MultiFactor 权重优化与风险平价

## MultiFactor 因子权重优化尝试

目标：改善 MultiFactor 子策略在真实数据回测中表现偏弱（CAGR ~6%，Sharpe ~0.36，MaxDD ~-17%）的问题。

### 已尝试并回测的方法

1. **更激进的静态因子权重**（提升 growth/momentum/行业动量/突破，降低 quality/lowvol/rsi_mr）  
   - 结果：组合和 MultiFactor 都变差。组合 CAGR 15.12% / Sharpe 1.009 / MaxDD -10.92%；MultiFactor CAGR 4.13% / Sharpe 0.261 / MaxDD -22.94%。
   - 已 revert。

2. **启用自适应 IC 权重**（在 `strategies/v14.py` 中调用 `run_v14` 时传入 `adaptive=True`）  
   - 结果：同样变差。组合 CAGR 14.97% / Sharpe 1.005 / MaxDD -9.76%；MultiFactor CAGR 3.76% / Sharpe 0.223 / MaxDD -20.86%。
   - 已 revert。

3. **将 MultiFactor 的 `weight_method` 从 `momentum_weighted` 改为 `risk_parity`**（仅改 `run_multi_strategy.py` 配置）  
   - 结果：显著改善风险指标。组合 CAGR 15.55% / Sharpe 1.060 / MaxDD -9.18% / Vol 14.67%；MultiFactor Sharpe 0.426 / MaxDD -13.40%（CAGR 略降至 5.72%）。
   - 已 push：commit `e0458d6` (`feat: switch MultiFactor to risk_parity weight method to reduce drawdown`)。

### 结论

直接调整 MultiFactor 内部 17 个因子的静态/自适应权重并未带来提升；通过 `risk_parity` 降低个股集中度，反而改善了整体组合和 MultiFactor 的风险调整收益。

## 操作教训

在 commit `run_multi_strategy.py` 时误用了 `git add -A`，导致大量未跟踪文件（`quant/`、`lean-root/` 等）被暂存。后续通过 `git reset --hard HEAD~1` 撤销错误 commit，重新单独 `git add run_multi_strategy.py` 并 push，确保只提交目标文件。以后 commit 前必须先用 `git status` 和 `git diff --cached` 检查暂存内容。

- **长期**：CAGR +0.08pp，Sharpe +0.017，MaxDD -0.01pp，Vol -0.15pp — 基本持平。

## 策略动量 TAA 尝试

为响应“优化算法”的方向，实现了基于策略动量的战术配置（TAA）：

- 实现位置：`weight_allocation.py` 新增 `MomentumTAA` 类；`strategies/portfolio.py` 在 `run_backtest` 中集成动态权重；`run_multi_strategy.py` 新增 `--dynamic-alloc` 等 CLI 参数。
- 算法逻辑：每个 rebalance 日（月末）回看过去 N 天，计算各子策略的 Sharpe（年化收益/年化波动），用 softmax 转换为权重，并加 40% 上限。
- 主窗口 2021-02-01 ~ 2023-12-29 真实数据回测对比：

| 配置 | CAGR | Sharpe | MaxDD | Vol |
|---|---|---|---|---|
| 固定权重 | 17.67% | 1.143 | -8.25% | 15.45% |
| TAA lookback 63 | 15.08% | 1.061 | -9.19% | 14.21% |
| TAA lookback 126 | 15.05% | 1.058 | -9.18% | 14.23% |
| TAA lookback 252 | 14.70% | 1.053 | -9.24% | 13.96% |

**结论**：当前 TAA 实现（Sharpe 动量 + 月度再平衡 + 40% 上限）在所有测试的 lookback 下均跑输固定权重，收益和夏普下降，仅波动率降低。说明该组策略的短期动量不具持续性，或月度再平衡过于频繁引入追涨杀跌。

**处理**：TAA 代码保留在本地工作区但不作为默认提交，待后续优化（如改用更长的季度再平衡、加入波动率趋势过滤、或改用 HRP/Regime 等算法）后再考虑合并。

## Hierarchical Risk Parity (HRP) 尝试

在 `WeightAllocator` 中实现了无 scipy 依赖的 HRP 方法：
- 基于相关系数距离矩阵做手动单链接层次聚类
- 沿聚类树递归二分，按子簇方差倒数分配权重

将各子策略的 `weight_method` 临时改为 `hrp` 后回测（主窗口 2021-02-01 ~ 2023-12-29）：

| 配置 | Final NAV | CAGR | Sharpe | MaxDD | Vol |
|---|---|---|---|---|---|
| 当前 G35 S30 + 0.20 | 1.8026 | 22.49% | **1.230** | -10.15% | 18.28% |
| HRP | 1.5782 | 17.01% | 1.015 | -8.54% | 16.76% |

HRP 跑输当前配置：CAGR -5.48pp，Sharpe -0.215。虽然波动和回撤更低，但收益损失过大。已 revert HRP 代码。
