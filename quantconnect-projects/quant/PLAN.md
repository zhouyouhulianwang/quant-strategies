### 阶段 2: 策略研究与开发

**目标：** 开发至少 3 个独立策略，每个策略包含完整文档

#### 2.1 策略开发规范

**每个策略必须包含：**
```python
class MyStrategy(QCAlgorithm):
    def initialize(self):
        """初始化：参数、指标、事件订阅"""
        self.set_start_date(2020, 1, 1)
        self.set_end_date(2024, 12, 31)
        self.set_cash(100000)
        self.set_benchmark("SPY")
        
        # 参数外部化（便于优化）
        self.lookback = self.get_parameter("lookback", 20)
        self.threshold = self.get_parameter("threshold", 0.02)
        
    def on_data(self, data: Slice):
        """主交易逻辑"""
        pass
    
    def on_order_event(self, order_event):
        """订单事件处理（日志记录）"""
        pass
```

#### 2.2 策略类型建议（初始组合）

| 策略 | 类型 | 资产类别 | 频率 | 风险等级 |
|------|------|----------|------|----------|
| 动量突破 | 趋势跟踪 | 美股 ETF | 日频 | 中 |
| 均值回归 | 统计套利 | 个股对 | 小时频 | 低 |
| 多因子选股 | 基本面量化 | 美股 | 月调仓 | 中 |

#### 2.3 研究流程

1. **Jupyter 研究**：`lean research <project>`
2. **数据探索**：检查数据质量、缺失值、异常值
3. **信号验证**：统计显著性检验（t-test, p-value < 0.05）
4. **过拟合检查**：
   - 训练/验证/测试集划分（60/20/20）
   - 参数稳健性检验
   - 不同市场环境测试（牛市/熊市/震荡）

### 阶段 3: 本地回测验证

**目标：** 验证策略在历史上的表现，识别潜在问题

#### 3.1 回测标准配置

```bash
# 回测命令
lean backtest "MyStrategy" \
  --start "2020-01-01" \
  --end "2024-12-31" \
  --output "results/backtest_001.json"
```

#### 3.2 回测检查清单

**必须验证的指标：**
- [ ] 总收益率 (Total Return)
- [ ] 年化收益率 (CAGR)
- [ ] 夏普比率 (Sharpe Ratio) > 1.0
- [ ] 最大回撤 (Max Drawdown) < 20%
- [ ] 胜率 (Win Rate)
- [ ] 盈亏比 (Profit Factor) > 1.5
- [ ] 索提诺比率 (Sortino Ratio)
- [ ] 卡尔玛比率 (Calmar Ratio)
- [ ] 交易次数（确保足够样本）
- [ ] 佣金/滑点影响

**过拟合检查：**
- [ ] 不同时间段的稳定性
- [ ] 参数敏感性分析
- [ ] 蒙特卡洛模拟
- [ ] 前后对比（walk-forward analysis）

#### 3.3 生成回测报告

```bash
lean report --backtest-results "results/backtest_001.json"
```

### 阶段 4: 参数优化

**目标：** 找到参数稳健区间，避免过拟合

#### 4.1 优化策略

```bash
# 参数优化示例
lean optimize "MyStrategy" \
  --parameter lookback:10:50:5 \
  --parameter threshold:0.01:0.05:0.01 \
  --target SharpeRatio
```

#### 4.2 优化原则

1. **避免过度优化**：参数组合 < 1000 种
2. **稳健区间验证**：最优参数附近的表现应该稳定
3. **OOS 验证**：优化参数在未见过数据上的表现
4. **时间衰减测试**：参数是否随时间失效

#### 4.3 优化后检查

- [ ] 最优参数不是边界值
- [ ] 参数曲面是平滑的（没有尖锐峰值）
- [ ] 不同随机种子结果一致

## 三、风险管理框架

### 3.1 风控层级

| 层级 | 机制 | 触发条件 | 动作 |
|------|------|----------|------|
| 1 | 策略内止损 | 单策略亏损 > 5% | 减仓/清仓 |
| 2 | 组合止损 | 总账户亏损 > 10% | 全部平仓 |
| 3 | 熔断 | 单日亏损 > 15% | 24h 禁止交易 |
| 4 | 人工干预 | 异常市场事件 | 手动停止 |

### 3.2 仓位管理规则

```python
# 凯利公式简化版
position_size = min(
    kelly_fraction * portfolio_value,
    max_position_pct * portfolio_value,  # 单策略上限 20%
    max_position_absolute                # 绝对金额上限
)
```
